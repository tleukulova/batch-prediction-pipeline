# Batch Prediction Pipeline - Titanic Survival Classifier

## Project Description

This project implements a **Batch Prediction Pipeline** for binary classification using the Titanic dataset.  
The pipeline reads passenger feature data from a SQLite database, applies a trained machine-learning model, and writes the survival predictions back into the database. The entire process runs automatically on a configurable schedule via a Python scheduler — no cron jobs required.

---

## Dataset Description

**Source:** [Titanic dataset](https://raw.githubusercontent.com/datasciencedojo/datasets/master/titanic.csv)

| Column | Type | Description |
|--------|------|-------------|
| Pclass | Integer | Passenger class (1 = first, 2 = second, 3 = third) |
| Sex | Text | Passenger gender (male / female) |
| Age | Float | Age in years (may contain missing values) |
| SibSp | Integer | Number of siblings / spouses aboard |
| Parch | Integer | Number of parents / children aboard |
| Fare | Float | Ticket fare paid |
| Embarked | Text | Port of embarkation (C, Q, or S) |

**Target column:** `Survived` (0 = did not survive, 1 = survived)  
The target is **not stored** in the `input_data` table — it is what the model predicts.

---

## Database Design

**Database:** `database/predictions.db` (SQLite)

### Table: `input_data`
Stores passenger features loaded from the Titanic CSV.

| Column | Type | Notes |
|--------|------|-------|
| id | INTEGER | Primary key, auto-incremented |
| Pclass | INTEGER | |
| Sex | TEXT | |
| Age | REAL | May be NULL (imputed at prediction time) |
| SibSp | INTEGER | |
| Parch | INTEGER | |
| Fare | REAL | |
| Embarked | TEXT | May be NULL (imputed at prediction time) |

### Table: `predictions`
Stores one prediction per input row.

| Column | Type | Notes |
|--------|------|-------|
| id | INTEGER | Primary key, auto-incremented |
| input_id | INTEGER | Foreign key → input_data.id (UNIQUE) |
| prediction | INTEGER | 0 = did not survive, 1 = survived |
| prediction_timestamp | TEXT | UTC timestamp of the prediction run |

The `UNIQUE` constraint on `input_id` ensures predictions are never duplicated.

---

## ML Model

**Algorithm:** `RandomForestClassifier` (scikit-learn)

**Preprocessing pipeline (ColumnTransformer):**
- Numeric features (`Pclass`, `Age`, `SibSp`, `Parch`, `Fare`): median imputation
- Categorical features (`Sex`, `Embarked`): most-frequent imputation + ordinal encoding

**Training:** 80/20 stratified train/test split with `random_state=42`  
**Hyperparameters:** `n_estimators=100`, `max_depth=6`, `class_weight="balanced"`

The full preprocessing + model pipeline is saved as `models/titanic_model.pkl` using Python's `pickle` module, ensuring identical transformations are applied at prediction time.

---

## Batch Prediction Pipeline Explanation

The pipeline (`batch_predict.py`) executes the following steps in each run:

1. **Connect** to `database/predictions.db`.
2. **Query** only input rows that have **no existing prediction** (via `LEFT JOIN`).
3. **Load** the trained model from `models/titanic_model.pkl`.
4. **Predict** survival for each unprocessed passenger.
5. **Write** results (prediction + UTC timestamp) into the `predictions` table.
6. **Disconnect** from the database.

If all rows already have predictions, the script exits safely with a message. Re-running is always idempotent.

---

## Scheduler Explanation

`scheduler.py` uses **APScheduler** (`BlockingScheduler`) to automate the pipeline:

- Runs one prediction immediately on startup.
- Repeats every **5 minutes** automatically.
- A single `Ctrl+C` stops the scheduler gracefully.
- `max_instances=1` ensures overlapping runs are prevented.

## How to Run the Project

### 1. Download the dataset

Download `titanic.csv` from:  
```
https://raw.githubusercontent.com/datasciencedojo/datasets/master/titanic.csv
```
Save it as `data/titanic.csv`.

### 2. Create and activate a virtual environment

```bash
python3 -m venv venv
source venv/bin/activate
```

### 3. Install dependencies

```bash
pip install -r requirements.txt
```

### 4. Create the database and load input data

```bash
python create_database.py
```

### 5. Train the model

```bash
python train_model.py
```

### 6. Run a single batch prediction (optional test)

```bash
python batch_predict.py
```

### 7. Start the scheduler (runs every 5 minutes automatically)

```bash
python scheduler.py
```

Press `Ctrl+C` to stop.
